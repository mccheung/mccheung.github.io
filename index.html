How to auto re-connect to ADSL pppoe in Debian

I am using Debian linux and I have a ADSL to connect on INTERNET. The ADSL will offline each 48 hour with ISP, So I need it can auto re-connect to INTERNET when it’s down. I’ll use pppoeconf to config the ADSL.

When done install support package, then

Just answer the series questions, forexample: username, password etc.

Now, I can connect from my terminal using: pon dsl-provider and disconnect using: poff -a.

Then I writed cron script to keep connect, if down, will reconnect

#!/bin/sh
# adsl-reconnect.sh

IFACE=ppp0
UP="pon dsl-provider"
LOG=/var/log/dsl-reconnect.log

PTP=`ifconfig $IFACE 2>&1|grep P-t-P|cut -d : -f 3|cut -d " " -f 1`
RECV=`ping -c 1 $PTP 2>&1|grep received|cut -d , -f 2|cut -d " " -f 2`

if [ "$RECV" != "1" ]
then
    echo "Try to poff connection... " >>$LOG
    `poff -a`

    for i in `ps -eaf | grep "pppd" | tr -s " " | cut -f2 -d " "`
    do
        kill -9 ${i}
    done

    `poff -a`
    sleep 3

    date>>$LOG
    echo "hopefully killed"
    $UP >>$LOG 2>&1
fi

Save it to /usr/bin/auto-reconnect.sh, then add to contab:

* * * * * /usr/bin/auto-reconnect.sh

All things done!!!

Reference:
AUTO RE-CONNECT TO DSL PPPOE IN LINUX
[Schedule Tasks on Linux Using Crontab](http://kvz.io/blog/2007/07/29/schedule-tasks-on-linux-using-crontab/

How to add a new hard disk on Debian OS

Adding a additional hard disk to your server is easy and often required.
Here’s steps through the process newly attached drive.

STEPS

1 Figure out the device name for the new device

root@debian:~# fdisk -l

Disk /dev/sdb: 1000.2 GB, 1000204886016 bytes
255 heads, 63 sectors/track, 121601 cylinders, total 1953525168 sectors
Units = sectors of 1 * 512 = 512 bytes
Sector size (logical/physical): 512 bytes / 4096 bytes
I/O size (minimum/optimal): 4096 bytes / 4096 bytes
Disk identifier: 0x00000000

**Disk /dev/sdb doesn't contain a valid partition table**

Disk /dev/sda: 128.0 GB, 128035676160 bytes
255 heads, 63 sectors/track, 15566 cylinders, total 250069680 sectors
Units = sectors of 1 * 512 = 512 bytes
Sector size (logical/physical): 512 bytes / 512 bytes
I/O size (minimum/optimal): 512 bytes / 512 bytes
Disk identifier: 0x00018dcc

   Device Boot      Start         End      Blocks   Id  System
/dev/sda1   *        2048   239855615   119926784   83  Linux
/dev/sda2       239857662   250068991     5105665    5  Extended
/dev/sda5       239857664   250068991     5105664   82  Linux swap / Solaris

`NOTE: Disk /dev/XXX  this usually is you new HDD Driver`

2 artion the new disk using the cfdisk

New -> Primary -> Specify size in MB -> Write -> yes
Quit

3 Format the new disk with ext4 filessystem

4 Create a new directory for mount the new disk in filesystem

5 Findout the UUID (Universally Unique Identifier)

/dev/sda5: UUID="44e8b854-8fe4-4a36-8d5a-f4cbed9536c6" TYPE="swap"
/dev/sdb1: UUID="8e3c7dd4-a4f8-434c-b74b-7b927dbabe73" TYPE="ext4"
/dev/sda1: UUID="b4ccfaeb-051e-4838-a082-e98fcaec078d" TYPE="ext4"</code>

6 Edit fstab to automatically mount new HDD on boot

`vim /etc/fstab`

**add below line at end of file:**

`UUID=8e3c7dd4-a4f8-434c-b74b-7b927dbabe73 /new_hdd ext4 errors=remount-ro 0 1`

7 You also can mount by manual( Optional) :

Now the new hard disk is mounted and ready to use.
